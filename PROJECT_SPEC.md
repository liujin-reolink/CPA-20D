# CPA 冲刺手册项目规范

## 1. 项目目标

本项目是一套面向 CPA 专业阶段的短期冲刺手册。

目标不是写教材，也不是做百科，而是帮助考生在考前约 30 天内快速建立知识体系、抓住高频考点、减少低效复习。

当前只做四门：

- 会计
- 审计
- 财务成本管理
- 经济法

暂不做：

- 税法
- 公司战略与风险管理

原因：这两门已通过。

## 2. 目标读者

默认读者是：

- 正在备考 CPA 的考生
- 有一定财务基础，但知识遗忘较多
- 时间有限，目标是通过考试
- 希望理解知识，而不是只背结论

目标分数区间：60-80 分。

## 3. 核心原则

每一句话都要服务于考试。

写作时始终问一句：

> 这句话能帮助考生理解、做题或避坑吗？

如果不能，删除。

## 4. 内容边界

本项目不追求完整覆盖教材。

保留：

- 高频考点
- 基础逻辑
- 考试常见问法
- 易错点
- 判断方法
- 必要公式和表格

减少或删除：

- 教材式长定义
- 准则背景和历史沿革
- 低频细节
- 学术化表述
- 大段法条原文

## 5. 写作风格

风格应像一位经验丰富的 CPA 老师在讲课。

要求：

- 专业，但不故作高深
- 清楚，但不啰嗦
- 先讲为什么，再讲是什么，最后讲怎么考
- 尽量使用短句
- 第一次出现专业词时，用人话解释

例如：

不要直接写：

> 企业应当在客户取得相关商品控制权时确认收入。

建议写：

> 收入不是收到钱就确认，而是客户真正拿到商品、能够决定怎么使用这件商品时确认。会计上把这叫“控制权转移”。

## 6. 每章固定结构

每章尽量采用以下结构：

```markdown
# 章节标题

> 重要程度：⭐⭐⭐⭐⭐
> 预计分值：
> 建议学习时间：

## 为什么会考这一章？

## 一句话理解

## 核心知识

## 考试重点

## 容易混淆

## 真题套路

## 一句话总结

## 考前一分钟
```

不要随意增加大量新栏目，避免结构变复杂。

## 7. 重要程度

统一使用五级：

| 等级 | 含义 |
|---|---|
| ⭐⭐⭐⭐⭐ | 必须掌握 |
| ⭐⭐⭐⭐ | 高频，建议熟练 |
| ⭐⭐⭐ | 中频，理解即可 |
| ⭐⭐ | 低频，知道即可 |
| ⭐ | 极低频，最后再看 |

## 8. 页数控制

双面打印时，每门考试不要超过 50 张纸。

换算为 A4 页数：

| 科目 | 页数上限 |
|---|---:|
| 会计 | 100 页 |
| 审计 | 90 页 |
| 财务成本管理 | 100 页 |
| 经济法 | 80 页 |

章节篇幅参考：

| 重要程度 | 建议篇幅 |
|---|---:|
| ⭐⭐⭐⭐⭐ | 6-8 页 |
| ⭐⭐⭐⭐ | 4-6 页 |
| ⭐⭐⭐ | 2-4 页 |
| ⭐⭐ | 1-2 页 |
| ⭐ | 0.5-1 页 |

宁可删减，也不要无限扩写。

## 9. 例子使用原则

优先使用容易理解的业务场景。

建议贯穿使用：

- 奶茶店
- 制造企业
- 普通销售业务
- 银行借款
- 购买设备

不要频繁切换复杂互联网公司案例，除非该案例明显有助于理解。

## 10. Git 提交规范

建议使用：

- `docs:` 项目说明、路线图、规范
- `feat:` 新增章节
- `improve:` 优化章节
- `fix:` 修正错误
- `refactor:` 重构章节结构

## 11. 当前开发顺序

第一阶段优先完成会计核心章节：

1. 会计到底在学什么
2. 收入
3. 金融工具
4. 长期股权投资
5. 企业合并
6. 合并财务报表
7. 所得税
8. 租赁

完成会计主干后，再推进审计、财管、经济法。
